# Contributing Guide

This repository follows a learning-first development workflow.

---

## Development Principles

- Build incrementally
- Test continuously
- Commit frequently
- Push regularly
- Document important changes
- Review before merging

---

## Branch Strategy

Main branch

```
main
```

Feature branches

```
feature/pdf-upload

feature/chunking

feature/chat-ui
```

---

## Commit Convention

Use Conventional Commits.

Examples:

```
feat:

fix:

docs:

refactor:

test:

style:

chore:
```

Example

```
feat: implement PDF upload API

fix: resolve CORS configuration

docs: update architecture notes
```

---

## Pull Request Checklist

Before merging:

- Code builds successfully
- No lint errors
- Feature tested
- Documentation updated
- Meaningful commit messages
- Code reviewed

---

## Coding Standards

- Keep components small.
- Prefer reusable code.
- Follow separation of concerns.
- Use meaningful names.
- Avoid unnecessary complexity.

---

## Documentation

Update documentation whenever:

- Architecture changes
- API changes
- Folder structure changes
- Deployment changes

---

## Goal

The objective is not only to build software but also to understand the engineering principles behind every implementation.
